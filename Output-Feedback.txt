2. Current Add New Help Tip form usage and issue

Current code (frontend/src/app/maintenance/help-tips/page.tsx): <RichTextEditor value={newItem.helpTipContent} onChange={(html) => setNewItem({ ...newItem, helpTipContent: html })} readOnly={false} showToolbar minHeight={140} ariaLabel="Help tip content" className="bg-white" />

Problem:

- onChange spreads and recreates the entire newItem object on every keystroke, forcing a broader component re-render. Although the editor has caret restoration, the broader page re-render plus grid layout updates cause cursor jump and Enter not creating new line in some cases.

3. Fix for Add New Help Tip form to match established pattern

Change only frontend/src/app/maintenance/help-tips/page.tsx:

- Introduce a local state variable for the editor value, e.g., newContent.
- Initialize it from newItem.helpTipContent.
- Bind RichTextEditor.value to newContent.
- In onChange, call setNewContent only (do not mutate newItem per keystroke).
- On create (handleCreate), read helpTipContent from newContent.
- After successful create, reset newContent to '' alongside newItem.

This mirrors the CustomerInfoSection pattern and limits re-render scope, preventing caret loss and allowing Enter to insert a new line.

Proposed minimal changes:

- Add: const [newContent, setNewContent] = useState(newItem.helpTipContent ?? '')
- Add useEffect to keep newContent in sync when newItem resets: useEffect(() => { setNewContent(newItem.helpTipContent ?? ''); }, [newItem.helpTipContent])
- Update RichTextEditor: value={newContent} onChange={(html) => setNewContent(html)} readOnly={false} showToolbar minHeight={140} ariaLabel="Help tip content" className="bg-white"
- Update handleCreate to use: const helpTipContent = newContent ?? ''
- After successful create: setNewItem({ form: '', topic: '', helpTipContent: '' }) setNewContent('')
