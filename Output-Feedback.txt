Analysis of DataTable and header placement:
1) How DataTable renders its section header:
- DataTable (frontend/src/components/table/DataTable.tsx) renders only a table element: a table with thead for column headers and tbody for rows. It does not include any “section header” bar like “Draft Completed.” That top bar is provided by the page component, not by DataTable.

2) Is the header inside or outside the overflow-x-auto div:
- Before this fix, the “Draft Completed” section header in frontend/src/app/review-queue/page.tsx sat above the DataTable and was not given a min-width tied to the table. When the table grew wider than the viewport and scrolled, the header bar stayed at the container width and did not span the table’s full width.

Chosen fix path:
- Option B is not applicable (DataTable has no prop to control an outer scroll wrapper for a section header).
- Implemented Option C: Wrapped the entire header + DataTable + footer tip together in a single overflow-x-auto container and ensured the header bar spans the table width via min-w-full, with an inline-block min-w-max wrapper so the header expands to match the table’s width. This guarantees header and table scroll together with a single horizontal scrollbar.
