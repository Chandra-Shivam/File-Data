Task 1 — Verification report for “My View” dropdown

1) Service function called when My View changes
- Location: frontend/src/app/review-queue/page.tsx
- Behavior: The dropdown is bound to local state only:
  - State: const [myView, setMyView] = useState<string>("my");
  - onChange: onChange={(e) => setMyView(e.target.value)}
- There is no useEffect that depends on myView, and no API call is triggered from this change.
- Conclusion: No service function is called when My View changes; it updates local state only.

2) Full API call used by the Review Queue page
- Frontend function: getReviewQueuePage(sampleId?: number)
  - Source: frontend/src/services/api/reviews.ts
  - Endpoint: GET /api/v1/reviews/queue[?sampleId=...]
  - Query parameters: optional sampleId only
- Backend flow:
  - Controller: backend/src/Casrr.Api/Controllers/ReviewController.cs
    - Action: [HttpGet("queue")] GetReviewQueuePage([FromQuery] int? sampleId, CancellationToken ct)
    - Resolves employee number (emp) from the auth principal, calls service: _svc.GetReviewQueuePageAsync(sampleId, emp, ct)
  - Service: backend/src/Casrr.Application/Services/ReviewService.cs
    - Method: GetReviewQueuePageAsync(int? sampleId, string requestedByEmpNumber, CancellationToken ct)
    - Calls repo: _repo.GetQueuePageAsync(sampleId, ct)
  - Repository: backend/src/Casrr.Infrastructure/SqlServer/SqlReviewRepository.cs
    - Method: GetQueuePageAsync(int? sampleId, CancellationToken ct)
      - Progress counts via EF Core: _db.Reviews.Where(Cancelled != true).OptionalFilter(Sample_id == sampleId)
        - Table: dbo.[02_CORE_02_Reviews]
      - Queue rows via ADO.NET: SELECT … FROM dbo.[02_CORE_02_Reviews] WHERE Cancelled = 0 AND (Sample_id = @sampleId OR @sampleId IS NULL)
        - Table: dbo.[02_CORE_02_Reviews]
      - Sample dropdown (last 50) via EF Core from Samples
        - Table: dbo.[02_CORE_01_Samples]
- Note: The resolved employee number is not used by the repository in this flow.

3) Is the My View filter parameter being sent to the backend, or client-side filtering?
- No My View parameter is sent to the backend. Only sampleId is sent.
- There is no client-side filtering bound to myView either (no filtering logic keyed off myView).
- Conclusion: My View currently has no effect on data; it is neither server- nor client-filtering.

4) Exact filter values in the UI
- Values defined in page.tsx:
  - "my" → "My View"
  - "team" → "My Team"
  - "dept" → "My Department"
- No other options are present.
