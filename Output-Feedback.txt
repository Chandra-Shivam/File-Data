FIX 1 — Email fields not updating when dropdown changes

Root cause: The email fields use Field component with 
defaultValue (uncontrolled) in edit mode. Calling 
changes.setField updates the save context but does NOT 
update the displayed value. Need local state for emails.

Step A — Add local state for email values:
const [selectedReviewerEmail, setSelectedReviewerEmail] = 
  useState<string>(a?.reviewerEmail ?? "")
const [selectedManagerEmail, setSelectedManagerEmail] = 
  useState<string>(a?.managerEmail ?? "")

Step B — Sync with server data:
Add to the existing sync useEffect:
useEffect(() => {
  if (a?.reviewerName) setSelectedReviewer(a.reviewerName)
  if (a?.managerName) setSelectedManager(a.managerName)
  if (a?.reviewerEmail) setSelectedReviewerEmail(a.reviewerEmail)
  if (a?.managerEmail) setSelectedManagerEmail(a.managerEmail)
}, [a?.reviewerName, a?.managerName, 
    a?.reviewerEmail, a?.managerEmail])

Step C — Update local email state when dropdown changes:
In Reviewer Select onChange, add:
  if (user) {
    setSelectedReviewerEmail(user.email)  // ← ADD THIS
    changes?.setField("reviewInfo", "reviewerEmail", user.email)
  }

In Manager Select onChange, add:
  if (user) {
    setSelectedManagerEmail(user.email)  // ← ADD THIS
    changes?.setField("reviewInfo", "managerEmail", user.email)
  }

Step D — Replace email Field rendering in edit mode:
Find the Reviewer Email rendering in edit mode and 
replace the Field component with a controlled input:

{isEditing ? (
  <div className="flex flex-col gap-1">
    <label className="text-xs text-slate-500 font-medium 
      uppercase tracking-wider">REVIEWER EMAIL</label>
    <input
      className="w-full h-9 border border-slate-300 
      rounded px-2 text-sm bg-white"
      value={selectedReviewerEmail}
      onChange={(e) => {
        setSelectedReviewerEmail(e.target.value)
        changes?.setField("reviewInfo", "reviewerEmail", 
          e.target.value)
      }}
    />
  </div>
) : (
  // keep existing mailto link rendering exactly as is
)}

Apply identical pattern for Manager Email using 
selectedManagerEmail and setSelectedManagerEmail and 
field name "managerEmail".

FIX 2 — Examiner In Charge still editable in edit mode
Root cause: Removing section/name from Field stops 
saving changes but still renders an editable input.

Replace the Examiner In Charge Field entirely with a 
hardcoded read-only input that never changes regardless 
of isEditing:

BEFORE:
<Field label="EXAMINER IN CHARGE" 
  value={a?.examinerInCharge ?? ""} />

AFTER:
<div className="flex flex-col gap-1">
  <label className="text-xs text-slate-500 font-medium 
    uppercase tracking-wider">EXAMINER IN CHARGE</label>
  <input
    className="w-full h-9 border border-slate-200 
    rounded px-2 text-sm bg-slate-100 text-slate-500 
    cursor-not-allowed"
    value={a?.examinerInCharge ?? ""}
    readOnly
  />
</div>

This always shows the disabled styling regardless of 
edit mode — no conditional rendering needed.
