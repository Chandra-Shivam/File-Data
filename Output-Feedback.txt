Report: Review Form — Review Info page

1) How “Reviewer Name” and “Manager” are rendered in view vs edit (exact JSX)

In ReviewInfoSection.tsx (Assignments card), both fields are rendered via the shared Field component:
- Reviewer Name (invocation)
  &lt;Field label="Reviewer Name" value={a?.reviewerName ?? ""} section="reviewInfo" name="reviewerName" /&gt;

- Manager (name) (invocation)
  &lt;Field label="Manager" value={a?.managerName ?? ""} section="reviewInfo" name="managerName" /&gt;

The Field component (frontend/src/app/review/[ecif]/review-info/components/ui.tsx) switches its JSX based on edit mode:

- Edit mode (isEditing === true)
  &lt;input
    className="w-full h-9 border border-slate-300 rounded px-2 text-sm bg-white"
    defaultValue={value ?? ""}
    onChange={(e) =&gt; {
      if (changes &amp;&amp; section &amp;&amp; name) {
        changes.setField(section, name, e.target.value);
      }
    }}
  /&gt;

- View mode (isEditing === false)
  &lt;input
    className="w-full h-9 border border-slate-200 rounded px-2 text-sm bg-slate-50"
    value={value ?? ""}
    readOnly
  /&gt;

2) Backend endpoint for CAS users and any frontend “fetch by role”

- Backend endpoint exists
  Controller: backend/src/Casrr.Api/Controllers/CasUsersController.cs
  Route prefix: [Route("api/v1/cas-users")]
  Endpoints implemented:
  - GET /api/v1/cas-users
  - GET /api/v1/cas-users/{id}
  - POST /api/v1/cas-users
  - PUT /api/v1/cas-users/{id}
  - DELETE /api/v1/cas-users/{id}
  Auth policy: [Authorize(Policy = "RequireActiveUser")]

- Frontend services “fetch users by role”
  File: frontend/src/services/api/casUsers.ts
  There is no function that fetches users by role (no role-filtered path or query). Only listUsers (GET /api/v1/cas-users) and CRUD by id.
