Implemented the scroll-unit fix strictly in frontend/src/app/review-queue/page.tsx, without touching parent layout or AppShell.

Findings summary (no parent changes made):
- Parent AppShell main sets overflow-x-hidden; we did not modify it.
- DataTable has no code that sets scrollLeft or manipulates horizontal scroll; no useEffect refs for scroll.
- review-queue/page.tsx previously had overflow-hidden on the section, which clips content before the inner overflow-x-auto can scroll.

Fix applied in review-queue/page.tsx:
- Removed overflow-hidden from the <section> to prevent clipping.
- Wrapped header + table + footer inside a single minWidth container within one overflow-x-auto so they scroll together with one horizontal scrollbar.
- Kept min-w-0 on the section so the grid cell can shrink when needed; overflow-x-auto handles horizontal scrolling.

Outcome:
- Header and table now share one width and scroll together inside a single overflow-x-auto container.
- Removing overflow-hidden from the section eliminates clipping of the scroll content.
- The section retains min-w-0 so it fills the grid cell and can shrink below content when necessary; the inner container handles horizontal scrolling and shows the correct portion.
- No backend changes were made.
