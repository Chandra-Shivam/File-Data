FIX 1 — Dropdown selection not reflecting visually
Root cause: The Select uses value={a?.reviewerName} 
which comes from server data and does NOT update when 
changes.setField is called. It needs local React state.

Add local state for both dropdowns:
const [selectedReviewer, setSelectedReviewer] = 
  useState<string>(a?.reviewerName ?? "")
const [selectedManager, setSelectedManager] = 
  useState<string>(a?.managerName ?? "")

Sync with server data when it loads:
useEffect(() => {
  if (a?.reviewerName) setSelectedReviewer(a.reviewerName)
  if (a?.managerName) setSelectedManager(a.managerName)
}, [a?.reviewerName, a?.managerName])

Update the Reviewer Select:
BEFORE: value={a?.reviewerName ?? ""}
AFTER:  value={selectedReviewer}

Add setSelectedReviewer to its onChange:
onChange={(e) => {
  setSelectedReviewer(e.target.value)
  const user = croUsers.find(u => u.name === e.target.value)
  changes?.setField("reviewInfo", "reviewerName", 
    e.target.value)
  if (user) {
    changes?.setField("reviewInfo", "reviewerEmail", 
      user.email)
  }
}}

Apply identical pattern for Manager Select using 
selectedManager and setSelectedManager.

FIX 2 — Examiner In Charge always read-only
Even in edit mode, Examiner In Charge must stay 
disabled and non-editable.

Find the Examiner In Charge Field and force it to 
always render as view mode by removing section and 
name props so the Field component never enters edit 
mode for this field:

BEFORE:
<Field label="Examiner In Charge" 
  value={a?.examinerInCharge ?? ""} 
  section="reviewInfo" 
  name="examinerInCharge" />

AFTER:
<Field label="EXAMINER IN CHARGE" 
  value={a?.examinerInCharge ?? ""} />

(Removing section and name prevents Field from 
registering changes — it always shows the read-only 
view regardless of isEditing)
