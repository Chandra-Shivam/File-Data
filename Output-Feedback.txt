STOP. Root cause identified — do NOT modify AppShell.tsx.
Only fix frontend/src/app/review-queue/page.tsx.

Two problems found:
1. AppShell's <main> has overflow-x-hidden — this 
   conflicts with inner overflow-x-auto containers
2. <section className="min-w-0 overflow-hidden"> — 
   overflow-hidden on the section clips content 
   before overflow-x-auto can scroll it

Fix: Remove overflow-hidden from section. Wrap ALL 
content (header + table + footer) inside a single 
min-width div inside overflow-x-auto so everything 
scrolls as one unit:

BEFORE:
<section className="min-w-0 overflow-hidden">
  <div className="overflow-x-auto">
    <div className="... min-w-[700px] w-full">
      Draft Completed
    </div>
    <DataTable className="w-full min-w-[700px]" />
    <div className="... min-w-[700px] w-full">
      Tip...
    </div>
  </div>
</section>

AFTER:
<section className="min-w-0">
  <div className="overflow-x-auto">
    <div style={{ minWidth: '700px' }}>
      <div className="px-3 py-2 bg-[#1F3864] text-white 
        text-xs font-semibold border-b border-slate-200">
        Draft Completed
      </div>
      <DataTable
        columns={columns}
        rows={sortedRows}
        sortBy={sortBy}
        sortDir={sortDir}
        onSort={handleSort}
        selectedId={selectedId}
        getRowId={(r) => r.id}
        onSelect={setSelectedId}
        onOpen={(r) => { void handleOpen(r); }}
        tableLabel="Review Queue"
        className="w-full"
        theadClassName="bg-[#1F3864] text-white"
        selectedRowClassName="bg-slate-100"
      />
      <div className="text-xs text-slate-500 px-3 py-2 
        border-t border-slate-200">
        Tip: Double-click a row or press Enter/Space to open.
      </div>
    </div>
  </div>
</section>

Why this works:
- overflow-hidden removed from section — no more 
  clipping before scroll can happen
- Single div with style={{ minWidth: '700px' }} wraps 
  ALL three elements — header, table, footer scroll 
  as one unit with identical width
- overflow-x-auto on the direct parent handles scrolling
- section keeps min-w-0 so grid cell can shrink below 
  content size — overflow-x-auto handles the rest
- Using style prop for minWidth avoids Tailwind 
  conflicts with w-full/min-w-full
