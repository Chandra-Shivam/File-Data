STOP. Only modify the 9 date inputs in:
frontend/src/app/review/[ecif]/review-info/components/
sections/ReviewInfoSection.tsx

The date inputs use value={toIsoDate(...)} which is 
controlled. When onChange fires and calls setField, 
React resets the input because the source value 
(s?.croStart etc.) hasn't changed.

Fix ALL 9 date inputs with this pattern:

BEFORE (controlled — broken):
<input
  type="date"
  value={toIsoDate(s?.croStart || "")}
  onChange={(e) => {
    changes?.setField("reviewInfo", "startDate", 
      e.target.value)
  }}
  className={`... ${toIsoDate(s?.croStart || "") 
    ? 'has-value' : ''}`}
/>

AFTER (uncontrolled with key — works):
<input
  type="date"
  key={toIsoDate(s?.croStart || "")}
  defaultValue={toIsoDate(s?.croStart || "")}
  onChange={(e) => {
    e.currentTarget.classList.toggle(
      'has-value', !!e.target.value)
    changes?.setField("reviewInfo", "startDate", 
      e.target.value)
  }}
  onBlur={(e) => {
    changes?.setField("reviewInfo", "startDate", 
      e.target.value)
  }}
  className={`w-full h-9 border border-slate-300 
    rounded px-2 text-sm bg-white 
    ${toIsoDate(s?.croStart || "") ? 'has-value' : ''}`}
/>

Why this works:
- defaultValue lets the browser manage display state
- key={toIsoDate(...)} forces remount when entering 
  edit mode so the input starts with the correct value
- classList.toggle('has-value') makes text visible 
  when user selects a date (overrides globals.css rule)
- onBlur as fallback for browsers that fire blur 
  instead of change on date selection
