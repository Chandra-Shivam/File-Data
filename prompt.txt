1) backend/src/Casrr.Api/Controllers/ReviewController.cs
- How the current user is identified and how emp/employee number is resolved:
  The controller inherits helpers from BaseTemplateController and calls ResolveEmpNumberAsync(User, ct) to resolve the employee number from the auth principal. Full resolver code (BaseTemplateController.ResolveEmpNumberAsync):

    protected async Task&lt;string&gt; ResolveEmpNumberAsync(ClaimsPrincipal? user, CancellationToken ct)
    {
        try
        {
            var me = _graphUserInfoProvider is null ? null : await _graphUserInfoProvider.TryGetGraphMeAsync(user ?? HttpContext?.User!, ct);

            if (me is not null)
            {
                // Prefer explicit EmployeeId
                if (!string.IsNullOrWhiteSpace(me.EmployeeId))
                    return me.EmployeeId!.Trim();

                // Fallback: SAM that looks like an Emp #
                var sam = me.OnPremisesSamAccountName;
                if (!string.IsNullOrWhiteSpace(sam))
                {
                    var s = sam!.Trim();
                    if (LooksLikeEmpNumber(s))
                        return s;
                }
            }
        }
        catch
        {
            // Swallow and continue to claim-based heuristics
        }

        // Heuristic claim-based fallback (dev-only)
        var empId = GetClaimIgnoreCase(user, "employeeid")
                 ?? GetClaimIgnoreCase(user, "employeeId")
                 ?? GetClaimIgnoreCase(user, "extension_EmployeeNumber")
                 ?? GetClaimIgnoreCase(user, "extension_employeeNumber")
                 ?? null;

        if (!string.IsNullOrWhiteSpace(empId))
            return empId.Trim();

        var samClaim = GetClaimIgnoreCase(user, "onpremisessamaccountname")
               ?? GetClaimIgnoreCase(user, "onPremisesSamAccountName")
               ?? GetClaimIgnoreCase(user, ClaimTypes.Upn)
               ?? GetClaimIgnoreCase(user, "preferred_username")
               ?? null;

        if (!string.IsNullOrWhiteSpace(samClaim))
        {
            var s = samClaim.Trim();
            if (LooksLikeEmpNumber(s))
                return s;
        }

        // Conservative fallback when identity cannot be resolved
        var devEmp = Environment.GetEnvironmentVariable("Casrr_DEV_EMP");
        if (!string.IsNullOrWhiteSpace(devEmp))
            return devEmp.Trim();
        return "CC058";
    }

2) backend/src/Casrr.Infrastructure/SqlServer/SqlReviewRepository.cs — GetQueuePageAsync
- Complete SQL query that returns queue rows (from the private helper GetQueueRowsAsync):

    SELECT TOP (250)
        r.[Review_id],
        r.[Sample_id],
        r.[eCIF_number],
        r.[Customer_name],
        r.[CRO_name],
        r.[Relationship_mgr_name],
        r.[Portfolio_mgr_name],
        r.[TTBA_exposure],
        r.[Bank_PD],
        r.[CAS_PD],
        r.[Completed_date],
        r.[Review_finalized_date],
        r.[Review_approval_date],
        r.[Review_distributed_date],
        r.[Start_date],
        agg.[CommittedExposure] AS [AccountsCommittedExposure]
    FROM dbo.[02_CORE_02_Reviews] AS r WITH (NOLOCK)
    LEFT JOIN (
        SELECT a.[Review_id], SUM(COALESCE(a.[Commitment], 0)) AS [CommittedExposure]
        FROM dbo.[02_CORE_04_Accounts] AS a WITH (NOLOCK)
        GROUP BY a.[Review_id]
    ) AS agg
        ON agg.[Review_id] = r.[Review_id]
    WHERE (r.[Cancelled] IS NULL OR r.[Cancelled] = 0)
      AND (@sampleId IS NULL OR r.[Sample_id] = @sampleId)
    ORDER BY
        ISNULL(r.[Review_approval_date], ISNULL(r.[Review_finalized_date], ISNULL(r.[Completed_date], ISNULL(r.[Review_distributed_date], ISNULL(r.[Start_date], '1900-01-01'))))) DESC,
        r.[Review_id] DESC;

  Mapping in code derives reviewer as the first non-empty of CRO_name, Relationship_mgr_name, Portfolio_mgr_name.

- Column that links the review to the reviewer:
  The reviewer display is derived from name fields on the Reviews header, prioritized as:
  - CRO_name (primary)
  - Relationship_mgr_name (fallback)
  - Portfolio_mgr_name (fallback)
  No reviewer ID/email is used for linkage in this query; it’s based on those name columns.

3) CAS Users table structure (Casrr.Domain/CasUsers.cs)
- Fields (relevant to the question):
  - Manager: string // CAS_user_manager
  - ManagerEmail: string // CAS_user_manager_email
- Confirmation about the manager field:
  The Manager field stores a display name string (e.g., "Geoffrey Houlditch"), not an ID or email. The ManagerEmail field stores the email address. The Id property on CasUser represents the Windows/Network ID (CAS_user_id), not the employee number.

5) frontend/src/services/api/reviews.ts
- getReviewQueuePage function signature and behavior:

    export async function getReviewQueuePage(sampleId?: number): Promise<ReviewQueuePageResponse> {
      const qs = sampleId !== undefined && sampleId !== null && !Number.isNaN(sampleId)
        ? `?sampleId=${encodeURIComponent(String(sampleId))}`
        : "";
      return await get<ReviewQueuePageResponse>(`/api/v1/reviews/queue${qs}`);
    }

  It accepts an optional numeric sampleId and calls GET /api/v1/reviews/queue with ?sampleId= when provided. The response type is ReviewQueuePageResponse.
