Findings after reviewing AGENTS.md and the specified files:

1) Backend: SqlReviewRepository.GetReviewByKeys (and equivalents)
- Expected fields and mappings:
  - Prior_CAS_review_id → priorReviewId: NOT returned. None of the ReviewHeader SELECTs include Prior_CAS_review_id, and ReviewHeader has no property for it. Not mapped into ReviewForm.ReviewInfo.
  - Prior_CAS_review_date → priorReviewDate: NOT returned. Not selected in any ReviewHeader SELECT and no property present. Not mapped into ReviewForm.ReviewInfo.
  - Review_initial_approval_date → initialApproval: NOT returned. The ReviewHeader SELECTs do not include Review_initial_approval_date and there is no property for it. Not mapped into ReviewForm.ReviewInfo.
  - Mentor_name → secondReviewer: NOT returned. The ReviewHeader SELECTs do not include Mentor_name and there is no property for it. Not mapped into ReviewForm.ReviewInfo.
  - Mentor_review_date → secondReviewDate: NOT returned. The ReviewHeader SELECTs do not include Mentor_review_date and there is no property for it. Not mapped into ReviewForm.ReviewInfo.
- Evidence:
  - GetReviewHeaderByIdAsync / GetLatestReviewHeaderForEcifAsync / GetLatestReviewHeaderForSampleAndEcifAsync SELECT lists do not contain the five columns above.
  - ReviewHeader class defines many fields, but none for Prior_CAS_review_id, Prior_CAS_review_date, Review_initial_approval_date, Mentor_name, Mentor_review_date.
  - ReviewForm.ReviewInfo is constructed without any prior/mentor/initial-approval fields.

Result: All 5 fields are missing from the backend payload.

Result: Only initialApproval exists in shape but is empty; the other four are not mapped.
