FIX 1 — Radio buttons disabled in Unlock modal
The radio inputs <input type="radio"> inside the 
unlock options modal are blocked by fieldset disabled.

Replace each radio option with a div-based toggle:
- Remove <input type="radio">
- Use a clickable <div> that calls the same handler
- Visual indicator: use a filled/empty circle div 
  instead of native radio

Pattern for each option:
<div
  role="radio"
  aria-checked={unlockReason === "GENERAL"}
  tabIndex={0}
  onClick={() => setUnlockReason("GENERAL")}
  onKeyDown={(e) => {
    if (e.key === 'Enter' || e.key === ' ')
      setUnlockReason("GENERAL")
  }}
  className="flex items-center gap-2 cursor-pointer 
    select-none p-2 rounded hover:bg-slate-50"
>
  <div className={`w-4 h-4 rounded-full border-2 
    flex items-center justify-center
    ${unlockReason === "GENERAL" 
      ? "border-sky-500 bg-sky-500" 
      : "border-slate-400"}`}>
    {unlockReason === "GENERAL" && (
      <div className="w-2 h-2 rounded-full bg-white" />
    )}
  </div>
  <span className="text-sm">
    Unlock review for general revisions
  </span>
</div>

Apply to all 3 radio options: GENERAL, 
RECONSIDERATION, APPEAL.
