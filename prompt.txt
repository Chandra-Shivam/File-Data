Reconsideration and Appeal modals in:
frontend/src/app/review/[ecif]/review-info/components/

The date picker shows but selected value doesn't 
persist. Root cause: value={rcDate} on a controlled 
date input resets when parent re-renders during 
date selection in a portal.

Fix: Change from controlled to uncontrolled 
for date inputs only. Use defaultValue + onChange 
+ ref pattern:

RECONSIDERATION modal date input:
BEFORE:
<input
  type="date"
  value={rcDate}
  onChange={(e) => setRcDate(e.target.value)}
  className="..."
/>

AFTER:
<input
  type="date"
  defaultValue={rcDate}
  onChange={(e) => setRcDate(e.target.value)}
  onBlur={(e) => setRcDate(e.target.value)}
  className="..."
/>

APPEAL modal date input:
BEFORE:
<input
  type="date"
  value={apDate}
  onChange={(e) => setApDate(e.target.value)}
  className="..."
/>

AFTER:
<input
  type="date"
  defaultValue={apDate}
  onChange={(e) => setApDate(e.target.value)}
  onBlur={(e) => setApDate(e.target.value)}
  className="..."
/>

Why this works:
- defaultValue sets the initial value but the 
  browser manages the visual state — no React 
  re-render can reset what the user typed/selected
- onChange still captures every change to state
- onBlur as fallback ensures value is captured 
  when user closes the date picker
